# 🖥️ 투민시스템 홈페이지 관리 및 운영 매뉴얼 (일반 관리자용)

이 문서는 개발자나 AI 어시스턴트(안티그래비티)의 도움 없이, **일반 관리자가 직접 텍스트 편집기와 윈도우 환경만 사용하여 홈페이지를 안정적으로 수정, 관리, 배포할 수 있도록** 수립된 지침서입니다.

## 📂 1. 홈페이지 구조와 작동 원리

투민시스템 홈페이지는 **정적 웹사이트(Static Website)**로 제작되었습니다. 데이터베이스 서버가 따로 없고, 모든 화면 정보와 제품 정보가 웹 문서(HTML) 안에 직접 기록되어 작동합니다.

* **호스팅 서비스 (서버)**: **GitHub Pages (깃허브 페이지)**
  * 비용이 완전히 무료이며, 트래픽 제한이 거의 없어 대량 접속 시에도 안정적입니다.
* **도메인**: `twominsystem.com` (폴더 내 `CNAME` 파일이 도메인 연결을 담당합니다.)
* **핵심 파일 및 폴더 구조**:
  * [index.html](file:///E:/업무/투민 홈페이지/홈페이지 자료/twoomin/index.html): 메인 페이지 (브랜드 소개, 제품 리스트, B2B 견적 문의 양식 탑재)
  * [product-detail.html](file:///E:/업무/투민 홈페이지/홈페이지 자료/twoomin/product-detail.html): 제품 상세페이지 (제품 코드 파라미터를 받아 정보를 동적으로 보여주는 공용 템플릿)
  * **이미지 파일들 (`.jpg`, `.png`)**: 웹사이트에 표시되는 제품 사진 및 인포그래픽 상세 이미지들 (폴더의 루트 경로에 직접 위치)
  * [publish.bat](file:///E:/업무/투민 홈페이지/홈페이지 자료/twoomin/publish.bat): 수정된 내용을 인터넷 서버에 반영해 주는 자동 배포 프로그램

## ✍️ 2. 홈페이지 내용 및 제품 정보 수정하기

파일을 수정할 때는 윈도우 기본 메모장 대신 **Notepad++**나 **VS Code** 같은 무료 코드 편집기를 사용하시는 것을 권장합니다. (글자 인코딩 깨짐을 방지하고 코드 줄 번호가 보여서 관리하기 쉽습니다.)

### ① 메인 페이지 글자 수정하기
1. [index.html](file:///E:/업무/투민 홈페이지/홈페이지 자료/twoomin/index.html) 파일을 편집기로 엽니다.
2. 바꾸고 싶은 글자(예: 주소, 전화번호 등)를 `Ctrl + F`로 검색하여 찾아 수정합니다.
3. 파일을 저장(`Ctrl + S`)합니다.

### ② 상세페이지 제품 데이터베이스(DB) 수정하기
상세페이지의 제품 스펙이나 상세 인포그래픽 이미지를 바꾸려면 [product-detail.html](file:///E:/업무/투민 홈페이지/홈페이지 자료/twoomin/product-detail.html) 파일 하단에 있는 **`productDb`** 자바스크립트 데이터를 수정해야 합니다.

> [!NOTE]
> `product-detail.html` 파일의 약 580번 라인부터 다음과 같은 형식의 **제품 데이터베이스**가 작성되어 있습니다.

## ⚠️ 3. 중요: 자바스크립트(JS) 문법 오류 예방 수칙

비개발자가 자바스크립트 코드(`productDb`)를 직접 편집할 때 가장 자주 범하는 실수를 예방하는 방법입니다. **쉼표 한 개만 누락되어도 페이지가 완전히 백지로 변하므로 아래 수칙을 반드시 지켜주셔야 합니다.**

1. **콤마(쉼표 `,`) 누락 주의**:
   * 각 속성의 끝에는 반드시 쉼표( `,` )를 붙여야 합니다. 
   * **잘못된 예**: `name: 'TA100'` (다음 줄이 있다면 끝에 쉼표가 없어서 오류 발생)
   * **올바른 예**: `name: 'TA100',`
2. **괄호의 짝 맞추기**:
   * 모든 중괄호 `{ }` 와 대괄호 `[ ]` 는 열린 만큼 닫혀야 합니다. 기존 제품 코드를 그대로 복사해서 수정할 때 괄호가 지워지지 않도록 주의하세요.
3. **따옴표 짝 맞추기**:
   * 글자를 감싸는 작은따옴표( `'` )나 큰따옴표( `"` )는 반드시 양쪽에 한 쌍으로 존재해야 합니다.
   * **잘못된 예**: `'TA100_detail_black.jpg` (끝나는 따옴표 누락)
   * **올바른 예**: `'TA100_detail_black.jpg'`

## 📋 4. 신규 제품 추가 단계별 가이드 (복사-붙여넣기 템플릿)

새로운 제품 모델(예: `TA400`)이 출시되어 상세페이지에 추가하고 싶을 때의 작업 단계입니다.

### 2단계: 이미지 파일 배치

### 3단계: 메인 화면 리스트에 링크 연결

   * **코드 예시**: `<div class="product-card" onclick="location.href='product-detail.html?model=TA400'">`

## 🔍 5. 자가 진단 및 오류 해결법 (F12 활용법)

코드를 수정한 뒤 홈페이지에 접속했는데 **화면이 완전히 하얗게 나오거나 정보가 로드되지 않는 경우** 해결 방법입니다.

1. 크롬(Chrome) 또는 엣지(Edge) 브라우저에서 홈페이지를 엽니다.
2. 키보드의 **`F12`** 버튼을 누릅니다 (개발자 도구 창이 뜹니다).
3. 상단 탭 중 **`Console (콘솔)`** 탭을 클릭합니다.
4. **빨간색 에러 메시지**를 확인합니다.
   * **예시**: `Uncaught SyntaxError: Unexpected token ... product-detail.html:629`
   * **해석**: `product-detail.html` 파일의 **629번째 줄** 주변에 쉼표( `,` )가 빠졌거나 따옴표 짝이 맞지 않는 문법 에러가 있다는 뜻입니다. 
   * 에러가 표시된 줄 번호로 찾아가 문법 수칙을 위반한 곳을 수정하면 간단히 해결됩니다.

## 🚀 8. 변경 사항 배포 및 깃허브(GitHub) 로그인 권한 오류 해결법

수정한 홈페이지를 반영할 때는 폴더 내 **`publish.bat`** 파일을 실행하면 되지만, 컴퓨터를 교체했거나 인증 토큰이 만료되면 업로드 실패 오류가 날 수 있습니다.

### ① 정상적인 업로드 순서
1. 파일 수정 및 이미지 파일 폴더 내 배치를 마칩니다.
2. **`publish.bat`** 파일을 더블클릭합니다.
3. 수정 내용을 간단히 작성하고 **Enter**를 누릅니다.
4. 1~2분 뒤 인터넷 창에서 **`Ctrl + Shift + R`** (강력 새로고침)을 눌러 반영 여부를 확인합니다.

### ② 로그인/권한 에러 발생 시 해결법 (`git push` 실패)
만약 검은색 도스 창에 **`Permission denied`** 또는 **`Log in to github.com`** 등 권한 관련 빨간 에러 메시지가 뜨며 배포가 실패하는 경우 조치법입니다.

1. 윈도우 시작 버튼 옆 검색창에 `cmd`를 입력하고 **명령 프롬프트**를 실행합니다.
2. 컴퓨터에 설치된 **GitHub CLI**를 이용해 로그인을 갱신합니다. 다음 명령어를 입력하고 엔터를 칩니다.
   ```bash
   gh auth login
   ```
3. 화면의 안내에 따라 키보드 방향키와 엔터로 옵션을 선택합니다.
   * **What account do you want to log in to?** -> `GitHub.com` 선택
   * **What is your preferred protocol for Git operations?** -> `HTTPS` 선택
   * **Authenticate Git with your GitHub credentials?** -> `Yes` 선택
   * **How would you like to authenticate GitHub CLI?** -> `Login with a web browser` 선택
4. 화면에 **8자리 일회용 코드 (예: `XXXX-XXXX`)**가 출력되며 브라우저 로그인 창이 자동으로 열립니다.
5. 브라우저 창에 8자리 코드를 입력하고 로그인 및 승인(Authorize) 버튼을 누르면 인증이 완료됩니다.
6. 다시 **`publish.bat`** 파일을 실행해 정상 업로드되는지 확인합니다.
